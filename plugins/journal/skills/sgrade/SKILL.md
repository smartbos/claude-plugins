# S등급 코칭 스킬

인사평가 S등급 달성을 위한 12가지 핵심역량 기반 회고 및 코칭.

## 트리거

- `/sgrade` 또는 `/회고` → 데일리 회고
- `/sgrade weekly` 또는 `/주간회고` → 주간 회고

## 시간대

**중요**: 모든 날짜/시간은 KST(한국 표준시, UTC+9) 기준.

```bash
TZ='Asia/Seoul' date '+%Y-%m-%d %H:%M %a'
```

## 파일 위치

| 유형 | 경로 |
|------|------|
| 저널 | `01.journals/연/월/일 요일.md` |
| 주간 회고 | `02.projects/셀프평가/주간/YYYY-W##.md` |
| 가이드 | `02.projects/셀프평가/셀프평가 가이드.md` |

요일 표기: 월, 화, 수, 목, 금, 토, 일

## 12가지 핵심역량

|  | 신의 (Trust) | 몰입 (Immersion) | 완결 (Completion) |
|---|---|---|---|
| **Idealist** | 관계 정렬 | 의미 몰입 | 의미 결과 |
| **Rational** | 사고 투명성 | 전략 실행 | 문제 구조화 |
| **Artisan** | 유연 협력 | 행동 편향 | 적응 해결 |
| **Guardian** | 신뢰 규율 | 지속 추진 | 완결 개선 |

## 현석님 역량 프로필

**강점 (유지/강화)**: 의미 몰입, 지속 추진, 유연 협력, 신뢰 규율

**성장 (의식적 개발)**: 행동 편향, 적응 해결, 문제 구조화, 사고 투명성

**스타일**: Idealist + Guardian 하이브리드. 의미를 이해하면 꾸준히 몰입해서 끝까지 간다.

**직급**: 수석 엔지니어 (Senior/Staff Engineer)

**조직 구성** (총 9명):
- CEO, CTO
- 서비스개발본부 (CTO 겸 본부장, 6명):
  - 시니어: 수석 엔지니어(나), 수석 디자이너
  - 미드레벨: 개발자 2명 (경력 3~4년)
  - 주니어: AI 엔지니어 1명 (인턴급)

> 소규모 조직이므로 수석 엔지니어의 역할 범위가 넓다. 아키텍처 의사결정, 주니어/미드레벨 성장 지원, CTO와의 기술 방향 조율이 모두 기대된다. 위임 대상이 제한적이므로 "무엇을 시스템화할 것인가"가 핵심 판단이다.

## 데일리 회고 프로세스

### 1단계: 저널 읽기
```bash
# KST 기준 오늘 날짜 확인
TZ='Asia/Seoul' date '+%Y %m %d %a'
```
해당 날짜의 저널 파일 읽기: `01.journals/연/월/일 요일.md`

### 2단계: 역량 평가
저널 내용을 12가지 역량 기준으로 분석:
- **발휘된 역량**: 저널에서 관찰된 행동/사건과 연결
- **개선 포인트**: 부족했던 역량과 구체적 코칭

### 3단계: 평가 기준

**강점 역량 체크**:
- 의미 몰입: 목적 이해하고 에너지 냈는가?
- 지속 추진: 꾸준히 집중했는가?
- 유연 협력: 동료 도움에 민첩하게 대응했는가?
- 신뢰 규율: 약속/기준 지켰는가?

**성장 역량 체크**:
- 행동 편향: 30% 정보로도 먼저 실행했는가?
- 적응 해결: 돌발 상황에 유연하게 대응했는가?
- 문제 구조화: 문제를 먼저 정의했는가?
- 사고 투명성: 사고 과정을 명확히 공유했는가?

**DON'T 체크**:
- 협업 대상자 관점 무시하고 단독 판단 안 했는가?
- 실행보다 분석/걱정에 시간 과도 사용 안 했는가?
- 마무리가 느슨하거나 미완 상태로 안 끝냈는가?

### 4단계: 결과 출력

사용자에게 평가 결과를 보여주고 확인 후 저널에 추가.

### 5단계: 저널에 추가

저널 파일 하단에 Daily Reflection 섹션 추가:

```markdown
---
## Daily Reflection

### 발휘된 역량
| 역량 | 근거 |
|------|------|
| (역량명) | (저널에서 관찰된 행동/사건) |

### 개선 포인트
| 역량 | 관찰 | 코칭 |
|------|------|------|
| (역량명) | (부족했던 점) | (구체적 개선 방법) |

### 한마디
(하루 요약 및 격려)
```

## 주간 회고 프로세스

### 트리거
- "주간 회고" → 코칭 플로우 (대화형)
- "주간 회고 간단히" → 분석 결과만 제시

### 1단계: 주간 저널 수집
```bash
# 이번 주 월~일 날짜 계산 (KST 기준)
TZ='Asia/Seoul' date '+%Y %m %d %a'
```
해당 주의 모든 저널 파일 읽기

### 2단계: 종합 분석
- 주간 역량 발휘 패턴
- 반복되는 강점/약점
- 성장 역량 발휘 빈도

### 3단계: 평가자 관점 분석

> **목적**: 역량 코칭이 "어떤 근육을 썼나"를 본다면, 평가자 관점은 "이 사람을 승진시킬 수 있나"를 묻는다. 12가지 역량 프레임만으로는 잡기 어려운 구조적 피드백을 제공한다.

**역할 기대치 기준 평가** (수석 엔지니어):

수석 엔지니어에게 기대되는 행동을 기준으로 이번 주 저널을 평가한다. 코칭 톤이 아니라 **인사평가 담당자가 실제로 작성할 법한 솔직한 평가** 톤으로 작성한다. 칭찬할 건 하되, 아쉬운 점은 구체적 근거와 함께 직설적으로 짚는다.

평가 관점:

| 관점 | 체크 포인트 |
|------|------------|
| **팀 레벨 임팩트** | 개인 실행을 넘어서 팀 전체에 영향을 미치는 행동이 있었는가? 아키텍처 방향 제시, 주니어 성장 지원, 프로세스 개선 등 |
| **위임 판단** | "내가 직접 안 해도 되는 일"과 "나만 할 수 있는 일"을 구분했는가? 반복 업무를 가이드/시스템으로 전환하려는 시도가 있었는가? |
| **시간 배분** | 수석 엔지니어로서 시간을 가장 높은 가치를 만드는 곳에 쓰고 있는가? 고객 대응, 기능 구현, 도구 최적화, 기술 리더십 간 균형은 적절한가? |
| **미완료 과제 관리** | 목표 대비 완료율은 어떠한가? 미완료 항목에 대한 위임, 일정 안내, 우선순위 재조정 판단이 있었는가? |
| **기술 리더십** | 기술적 의사결정을 주도했는가? 팀의 기술 부채를 인식하고 개선하려는 움직임이 있었는가? |

**종합 등급 판정**:
- 이번 주 전체를 업적평가 관점에서 S/A/B+/B/C 중 하나로 판정
- 판정 근거를 2~3문장으로 제시
- S등급과의 갭을 구체적으로 명시

**S등급으로 가기 위한 구조적 제언**:
- 역량 코칭(개선 포인트)과 별개로, 업무 구조/습관/역할 수행 차원의 제언 1~2개 제시

### 4단계: 이전 주간 회고 대비 성장 추적

이전 주간 회고 파일(`02.projects/셀프평가/주간/`)을 읽어서:
- 지난 주 "다음 주 포커스"로 제시한 항목의 달성 여부 확인
- 반복되는 개선 영역이 실제로 나아지고 있는지 추적

### 5단계: 주간 회고 파일 생성

`02.projects/셀프평가/주간/YYYY-W##.md` 형식으로 기록.

파일에 포함할 섹션:
1. 이번 주 핵심 요약 (한 문장)
2. 강점 역량 발휘 ✓
3. 성장 역량 발휘 ★
4. 개선 필요 영역
5. DON'T 체크
6. **평가자 관점** (종합 등급, S등급 갭, 구조적 제언)
7. 업적평가 관점 (S등급 체크)
8. 이전 주 대비 성장 포인트
9. 다음 주 포커스
10. 한마디

## 예시

```

```
/주간회고
```
→ 이번 주 저널들 종합 분석 및 코칭 + 평가자 관점 분석

```
